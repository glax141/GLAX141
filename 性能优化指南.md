# 🚀 图片性能优化指南

## 问题原因

当上传大量图片后，网站加载缓慢的主要原因：

| 问题 | 原因 | 影响 |
|------|------|------|
| 图片体积大 | Base64 编码比原图大约 33% | 页面加载慢 |
| 同时加载多张 | 所有缩略图一起加载 | 带宽占用高 |
| 没有懒加载 | 滚动时所有图片都渲染 | 内存占用大 |
| 换页等待长 | 新图片需要重新加载 | 用户体验差 |

---

## ✅ 解决方案

### 方案 1：使用外部图床（强烈推荐）

**最佳实践：图片使用 CDN 链接，不要上传 Base64**

#### 推荐的免费图床：

1. **jsDelivr + GitHub**（最推荐）
   - 上传图片到 GitHub 仓库
   - 使用链接：`https://cdn.jsdelivr.net/gh/用户名/仓库@main/图片.jpg`
   - ✅ 免费、全球 CDN、国内可访问

2. **Cloudinary**
   - 注册：https://cloudinary.com
   - ✅ 免费额度大、自动优化、支持变换

3. **ImgBB**
   - 上传：https://imgbb.com
   - ✅ 免费、简单、直接链接

#### 使用步骤：

1. 将图片上传到图床
2. 复制图片链接
3. 在后台管理 → 项目管理 → 直接粘贴 URL
4. 点击"保存到本地"

**优势：**
- 网站体积极小（只存 URL，不存图片数据）
- 加载速度快（专业 CDN）
- 跨设备同步快（数据量小）

---

### 方案 2：优化上传设置

如果必须使用 Base64，请修改上传参数：

在 `src/App.tsx` 中找到 `compressImage` 函数，修改：

```typescript
// 原来：maxWidth = 1920, quality = 0.85
// 修改为：
const compressImage = async (file: File, maxWidth = 1200, quality = 0.6): Promise<string> => {
  // ...
};
```

**效果：**
- 图片体积减少 60-70%
- 加载速度提升 2-3 倍

---

### 方案 3：减少每页图片数量

在后台管理的项目设置中，修改分页大小：

```typescript
const imagesPerPage = 10;  // 改为 6 或 8
```

---

### 方案 4：手动添加懒加载

由于修改代码需要重新构建，这里提供临时解决方案：

**在后台上传图片时：**
1. 每次只上传 5-10 张图片
2. 多个项目分散图片
3. 使用"下一页"功能查看

## ❓ 常见问题

**Q: 为什么上传后其他设备看不到？**
A: 因为数据保存在本地。请点击"同步到云端"按钮。

**Q: 图片太多导致同步失败？**
A: Gist 有大小限制。请使用 CDN 链接方案。

**Q: 如何批量替换图片链接？**
A: 下载 config.ts → 文本编辑器批量替换 → 上传到 GitHub
