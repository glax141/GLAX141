# 🔧 GitHub Gist 同步故障排除指南

## 问题：Token 配置后无法同步

### ✅ 检查清单（请按顺序检查）

#### 1. 检查 Token 权限

1. 打开 https://github.com/settings/tokens
2. 找到您的 `GLAX Website Sync` Token
3. 点击编辑（或重新创建）
4. **确保勾选了 `gist` 权限** ✅
5. 保存后重新复制 Token

**常见错误：**
- ❌ 没有勾选任何权限
- ❌ 只勾选了 `repo` 但没勾选 `gist`
- ✅ 必须勾选 `gist` 权限

---

#### 2. 检查 Token 格式

正确的 GitHub Token 格式：
- 长度：40 个字符（classic token）
- 格式：`ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

**常见错误：**
- ❌ 复制时多了空格
- ❌ 只复制了部分
- ❌ 复制了 `ghp_` 前面的内容

**解决方法：**
1. 删除 Token 输入框的所有内容
2. 重新从 GitHub 复制完整的 Token
3. 粘贴到网站后台
4. 点击"保存 Token"

---

#### 3. 测试 Token 是否有效

1. 登录网站后台
2. 输入 Token
3. 点击"保存 Token"
4. **查看状态提示**：
   - ✅ `Token 有效！已验证用户：glax141` - 成功
   - ❌ `Token 无效：Bad credentials` - Token 错误或过期
   - ⚠️ `Token 缺少 gist 权限` - 需要重新创建带 gist 权限的 Token
   - ❌ `无法连接 GitHub` - 网络问题

---

#### 4. 检查网络连接

GitHub API 需要能访问 `https://api.github.com`

**测试方法：**
1. 打开浏览器
2. 访问 https://api.github.com
3. 如果能看到 JSON 数据，说明网络正常

**如果无法访问：**
- 检查是否需要代理/VPN
- 尝试切换网络
- 稍后再试（可能是临时网络问题）

---

#### 5. 查看详细错误信息

1. 按 `F12` 打开浏览器开发者工具
2. 点击 **Console（控制台）** 标签
3. 点击"同步到云端"按钮
4. 查看控制台输出的错误信息

**常见错误：**
```
❌ 401 Unauthorized - Token 无效或权限不足
❌ 403 Forbidden - Token 没有 gist 权限
❌ 422 Unprocessable Entity - 数据格式错误
❌ Network Error - 网络连接问题
```

---

### 🔄 完整配置流程

#### 步骤 1：创建正确的 Token

1. 访问 https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**
3. 填写：
   - **Note**: `GLAX Website Sync`
   - **Expiration**: `No expiration`
   - **勾选**: ✅ `gist`（必须！）
4. 点击 **Generate token**
5. **立即复制 Token**（只显示一次！）

#### 步骤 2：配置网站后台

1. 打开网站 https://glax141.github.io/GLAX141/
2. 点击右上角 **Admin**
3. 输入密码 `glax141`
4. 找到 **云端同步设置** 区域
5. 粘贴 Token
6. 点击 **保存 Token**
7. 等待验证结果：
   - ✅ 看到 `Token 有效！用户：glax141 | 权限：gist`
   - ❌ 看到错误信息 → 查看上面的故障排除

#### 步骤 3：同步数据

1. 修改内容（图片、文字等）
2. 点击 **🟢 保存到本地**
3. 点击 **🔵 同步到云端**
4. 等待提示：
   - ✅ `Gist 创建成功！ID: xxxxx`
   - ✅ `同步成功！所有设备刷新页面即可看到更新`

#### 步骤 4：验证其他设备

1. 用手机或其他电脑打开网站
2. 刷新页面
3. 应该能看到最新的内容

---

### 🆘 仍然无法解决？

#### 方法 1：删除并重新创建 Token

1. 在 GitHub Token 页面删除现有 Token
2. 重新创建一个新的 Token（确保勾选 `gist`）
3. 在网站后台重新配置

#### 方法 2：使用本地保存（不跨设备）

如果 Gist 同步一直有问题，可以只用本地保存：

1. 修改内容
2. 点击 **🟢 保存到本地**
3. 刷新页面数据不会丢失
4. ⚠️ 但其他设备看不到更新

#### 方法 3：手动更新配置文件

1. 在后台修改内容
2. 点击 **下载 config.ts**
3. 上传到 GitHub 仓库（替换 `src/config.ts`）
4. GitHub Actions 会自动构建部署
