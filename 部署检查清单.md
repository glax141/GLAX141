## ✅ GitHub Pages 设置检查

### 步骤 1：确认仓库可见性

- [ ] 仓库是 **Public**（公开）或 **Private**（私有）都可以
- [ ] 如果是 Private，需要启用 GitHub Pages

### 步骤 2：设置 Pages 来源

1. 打开仓库 → **Settings**
2. 左侧菜单找到 **Pages**
3. **Source** 下拉菜单选择：
   - ✅ **GitHub Actions** （正确）
   - ❌ Deploy from a branch （错误！）

### 步骤 3：验证 Actions 权限

1. Settings → Actions → General
2. 确保 **Allow all actions and reusable workflows** 已启用
3. 如果是 Organization 仓库，检查组织设置

## 🔧 常见问题解决

### 问题 1：Actions 一直显示黄色（不运行）

**解决方法：**
1. Settings → Actions → General
2. 确保 Actions 已启用
3. 如果是 Fork 的仓库，需要手动启用 Actions

### 问题 2：部署成功但网站空白

**可能原因：**
- 浏览器缓存问题
- 网络问题
- JavaScript 错误

**解决方法：**
1. 按 `Ctrl + Shift + R`（Windows）或 `Cmd + Shift + R`（Mac）强制刷新
2. 清除浏览器缓存
3. 按 `F12` 打开开发者工具，查看 Console 错误

### 问题 3：404 Not Found

**可能原因：**
- Pages 设置错误
- 构建失败
- 路由问题

**解决方法：**
1. 确认 Pages Source 设置为 "GitHub Actions"
2. 查看 Actions 日志确认构建成功
3. 检查 `public/404.html` 是否存在

### 问题 4：图片无法加载

**可能原因：**
- 图片链接失效
- 跨域问题
- 网络问题

**解决方法：**
1. 使用可靠的图床（如 jsDelivr CDN）
2. 检查图片链接是否能直接访问
3. 在后台重新上传图片

## 📸 图片上传最佳实践

### 推荐方案：jsDelivr CDN

1. 创建一个专门的图片仓库（如 `GLAX-images`）
2. 上传图片到该仓库
3. 使用以下格式链接：
   ```
   https://cdn.jsdelivr.net/gh/glax141/GLAX-images@main/照片名.jpg
   ```

### 优点：
- ✅ 全球 CDN 加速
- ✅ 免费使用
- ✅ 国内访问速度快
- ✅ 稳定可靠

## 🎯 快速部署流程（3 分钟搞定）

1. **上传文件**（1 分钟）
   - 上传所有源代码文件
   - 在 GitHub 网页创建 `.github/workflows/deploy.yml`

2. **设置 Pages**（30 秒）
   - Settings → Pages
   - Source 选择 "GitHub Actions"

3. **等待部署**（2 分钟）
   - Actions 标签查看进度
   - 变绿后即可访问
